## 2. Kiến trúc dữ liệu (Database Schema - Supabase)

### A. Bảng `posts` (Quản lý bài viết)
*   `id`: (Primary Key) Định danh bài viết.
*   `title`: Tiêu đề bài viết.
*   `excerpt`: Đoạn trích dẫn ngắn (mô tả nhanh).
*   `content`: Nội dung chi tiết bài viết.
*   `category`: Nhãn chuyên mục (AI, Programming, TechReviews, Insights).
*   **`image_key`**: Lưu đường dẫn (file path) ảnh trong Supabase Storage (Ví dụ: `posts/174705-ai.jpg`).
*   `slug`: Đường dẫn thân thiện (ví dụ: `bai-viet-moi-ve-ai`).
*   `published`: Boolean (Mặc định: false) - Trạng thái xuất bản.
*   `created_at`: Thời gian tạo bài.
*   `updated_at`: Thời gian cập nhật bài viết gần nhất.

### B. Bảng `comments` (Quản lý bình luận)
*   `id`: Mã định danh bình luận.
*   **`post_id`**: Foreign Key liên kết với `posts.id`. 
    *   *Ràng buộc:* **ON DELETE CASCADE** (Khi xóa bài viết, tất cả bình luận liên quan sẽ tự động bị xóa).
*   `user_name`: Tên người bình luận.
*   `comment_body`: Nội dung bình luận.
*   `created_at`: Thời gian bình luận.

### C. Bảng `site_content` (Quản lý nội dung tĩnh)
Dùng cho các thông tin cố định nhưng muốn thay đổi linh hoạt như Banner, Hotline, Email, Footer...
*   `key`: (Unique) Tên vị trí (ví dụ: `hero_title`, `contact_phone`).
*   `value`: Nội dung hiển thị thực tế (văn bản hoặc link ảnh).
*   `updated_at`: Thời gian cập nhật gần nhất.

## 4. Luồng xử lý hình ảnh (Image Upload Flow)

Quy trình đăng bài kèm ảnh:
1.  **Người dùng chọn ảnh:** Admin chọn file từ máy tính trong Form.
2.  **Upload lên Storage:** Hệ thống upload file lên bucket `blog-images`.
3.  **Nhận File Path:** Supabase trả về đường dẫn file (ví dụ: `posts/my-image.jpg`).
4.  **Lưu vào Database:** Lưu đường dẫn này vào trường `image_key` trong bảng `posts`.

## 6. Cơ chế hiển thị & Dự phòng (Display Logic)

Sử dụng `.maybeSingle()` để tránh lỗi khi database chưa có dữ liệu cho key tương ứng.

```javascript
async function loadSiteContent(key, elementId) {
    const { data, error } = await supabase
        .from('site_content')
        .select('value')
        .eq('key', key)
        .maybeSingle(); // Tránh lỗi nếu chưa có dữ liệu

    if (data && data.value) {
        document.getElementById(elementId).innerText = data.value;
    } else {
        console.log(`Sử dụng nội dung mặc định cho: ${key}`);
    }
}
```

## 8. SQL Migration Script (Tạo bảng bằng code)

Dán đoạn mã này vào **SQL Editor** trong Supabase để khởi tạo toàn bộ hệ thống:

```sql
-- 1. Tạo bảng posts
CREATE TABLE posts (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  excerpt TEXT,
  content TEXT,
  category TEXT,
  image_key TEXT,
  slug TEXT UNIQUE,
  published BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. Tạo bảng site_content
CREATE TABLE site_content (
  key TEXT PRIMARY KEY,
  value TEXT,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 3. Tạo bảng comments
CREATE TABLE comments (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  post_id BIGINT REFERENCES posts(id) ON DELETE CASCADE,
  user_name TEXT NOT NULL,
  comment_body TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 4. Trigger tự động cập nhật updated_at
CREATE OR REPLACE FUNCTION update_modified_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_posts_modtime BEFORE UPDATE ON posts FOR EACH ROW EXECUTE PROCEDURE update_modified_column();
CREATE TRIGGER update_site_content_modtime BEFORE UPDATE ON site_content FOR EACH ROW EXECUTE PROCEDURE update_modified_column();
```
